## 三、SPA
### 设计意义
1. 前后端分离
2. 减轻服务器的压力：单页面，没有了多页面跳转重新下载的过程，只关注于API接口的调用
3. SPA SEO的问题

### 工作原理
1. history API
	window.history.pushState:向栈顶写入一个url去更新页面，replaceState去更新栈顶的记录即更新页面，popstate去控制页面的前进和后退
2. hash
	给window.onhanshchange函数绑定处理事件：获取到当前的网页地址的hash值，然后根据hash值进行渲染

#### 关键词：记录Ajax刷新，本地路由
